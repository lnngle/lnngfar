# p1 AI Skills 工作台

本目录提供面向多 AI Code 工具（Copilot、Qoder、CodeBuddy、Trae 等）的统一开发规范。

## 目标

- 统一上下文：避免不同 AI 输出互相冲突。
- 统一边界：避免误改 Cocos 关键目录与生成目录。
- 统一验收：所有 AI 改动都以 `npm run test` 为最低门禁。

## 快速使用

1. 先阅读 `context/`，理解项目与架构边界。
2. 再阅读 `policies/`，确认可改目录和禁改目录。
3. 按任务选择 `skills/` 下对应技能执行。
4. 输出必须包含可执行验证命令和回滚点。

## 目录说明

- `context/`：项目背景、架构与技术栈。
- `policies/`：目录权限、质量门禁、安全规则。
- `skills/`：12 个可组合的开发技能。
- `templates/`：任务卡、设计说明、测试计划模板。
- `adapters/`：不同 AI 工具的最小接入指引。

## 通用完成标准

- 代码可读、边界清晰、无无关改动。
- `npm run test` 通过。
- 涉及脚本改动时，补跑 `npm run test:unit`。
- 输出包含“影响目录 + 风险 + 回滚方案”。
