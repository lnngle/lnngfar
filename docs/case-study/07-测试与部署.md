### Repair Engine 自动修复

```bash
# 启动自动修复
$ lnngfar repair --auto

[Repair #1] 后端测试失败
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
问题：BorrowRecordServiceImpl.returnEquipment() 权限校验有问题

分析：
- 测试用例尝试用非借用人归还，应该抛出异常
- 但当前代码直接执行了归还操作

修复方案：
在 returnEquipment 方法中添加用户校验：

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void returnEquipment(Long recordId, Long userId, String remark) {
    BorrowRecord record = borrowRecordMapper.selectById(recordId);
    if (record == null) {
        throw new BusinessException("借用记录不存在");
    }
    
    // ⚠️ 修复：添加非借用人校验
    if (!record.getUserId().equals(userId)) {
        throw new BusinessException("非借用人不能操作归还");
    }
    
    // ⚠️ 修复：添加状态校验
    if (record.getStatus() != BorrowStatus.IN_USE.getCode()) {
        throw new BusinessException("设备不在使用中状态");
    }
    
    // ... 后续代码
}
```

### 生成的 docker-compose.yml

```yaml
# docker/docker-compose.yml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: equipment-mysql
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: equipment_db
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./mysql/init.sql:/docker-entrypoint-initdb.d/init.sql
    networks:
      - equipment-net

  redis:
    image: redis:7-alpine
    container_name: equipment-redis
    ports:
      - "6379:6379"
    networks:
      - equipment-net

  backend:
    image: equipment-backend:1.0.0
    container_name: equipment-backend
    environment:
      SPRING_PROFILES_ACTIVE: prod
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/equipment_db
      SPRING_REDIS_HOST: redis
    depends_on:
      - mysql
      - redis
    ports:
      - "8080:8080"
    networks:
      - equipment-net

  frontend:
    image: equipment-frontend:1.0.0
    container_name: equipment-frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - equipment-net

volumes:
  mysql_data:

networks:
  equipment-net:
    driver: bridge
```

## lnngfar deploy 功能需求清单

### 核心能力

| 能力 | 说明 | 实现方式 |
|------|------|----------|
| **Docker 构建** | 自动构建 Docker 镜像 | Docker API |
| **Compose 生成** | 生成 Docker Compose 配置 | Template Engine |
| **K8s 配置生成** | 生成 Kubernetes 部署配置 | Template Engine |
| **CI/CD 生成** | 生成 GitLab/GitHub Actions 配置 | Template Engine |
| **健康检查** | 验证服务部署状态 | HTTP Client |
| **一键部署** | 本地 Docker Compose 部署 | Shell |

## 完整部署命令

| 命令 | 说明 |
|------|------|
| `lnngfar test` | 运行所有测试 |
| `lnngfar test --module=backend` | 只运行后端测试 |
| `lnngfar test --module=frontend` | 只运行前端测试 |

| `lnngfar deploy docker-compose` | 生成 Docker Compose |
| `lnngfar deploy k8s` | 生成 K8s 配置 |
| `lnngfar deploy local` | 本地部署 |
| `lnngfar deploy ci` | 生成 CI/CD 配置 |
