# 02 - 项目初始化

> 本章演示如何使用 lnngfar 初始化一个完整的设备管理系统项目。

---

## 场景描述

开发团队（3人：1后端 + 1前端 + 1移动端）开始新项目，需要：

1. 创建前后端分离项目结构
2. 配置统一的技术栈
3. 初始化 Git 仓库
4. 制定开发规范

---

## 使用 lnngfar 初始化项目

### 步骤 1：检查环境

```bash
# 检查 lnngfar CLI 是否正确安装
$ lnngfar --version
lnngfar version 0.1.0

# 检查 Node.js (前端开发环境)
$ node --version
v20.11.0

# 检查 Java (后端开发环境)
$ java --version
openjdk 21.0.2

# 检查 Maven
$ mvn --version
Apache Maven 3.9.6
```

### 步骤 2：创建新项目

```bash
# 创建项目目录并进入
$ mkdir equipment-system && cd equipment-system

# 初始化项目（交互式）
$ lnngfar init

? 项目名称: equipment-system
? 项目描述: 公司设备管理系统
? 前端技术栈: (Use arrow keys)
  ❯ Vue3 + TypeScript + Element Plus
    React + TypeScript + Ant Design
    Vue3 + TypeScript + uni-app (小程序)
? 后端技术栈: (Use arrow keys)
  ❯ Spring Boot + MyBatis Plus
    Spring Boot + JPA
    NestJS
? 数据库: (Use arrow keys)
  ❯ MySQL 8.0
    PostgreSQL
    H2 (开发测试)
? 是否启用 uni-app 小程序: Yes
? 项目包名: com.company.equipment
? 基础包名: com.company
```

### 步骤 4：初始化 Git

```bash
# Git 已自动初始化
$ git status
On branch main

Untracked files:
  backend/
  frontend/
  uniapp/
  .ai/
  specs/
  docs/

# 添加 .ai/ 到 .gitignore（AI 相关文件不提交）
$ echo ".ai/prompts/" >> .gitignore
$ echo ".ai/memory.md" >> .gitignore
$ echo ".ai/.cache/" >> .gitignore

# 提交初始结构
$ git add .
$ git commit -m "feat: 初始化项目结构

- backend: Spring Boot + MyBatis Plus 后端项目
- frontend: Vue3 + TypeScript + Element Plus 前端项目
- uniapp: uni-app 小程序项目
- .ai: AI 上下文目录（不提交）
- specs: 需求规格文档
- docs: 项目文档"
```

## lnngfar 初始化功能需求清单

通过这个案例，我们发现 **lnngfar init** 需要具备以下能力：

### 核心能力

| 能力 | 说明 | 实现方式 |
|------|------|----------|
| **交互式创建** | 支持多种技术栈选择 | Inquirer.js 交互 |
| **模板生成** | 根据配置生成项目结构 | Template Engine |
| **技术栈组合** | 支持前后端+小程序组合 | 组合模板 |
| **配置持久化** | 保存项目配置到 lnngfar.config.json | 配置文件 |

### 模板引擎需求

| 模板类型 | 内容 |
|----------|------|
| **backend-template** | Spring Boot 项目基础结构 |
| **frontend-template** | Vue3 + Vite 项目结构 |
| **uniapp-template** | uni-app 项目结构 |
| **common-template** | 公共模块（utils, types等） |

### 配置项

| 配置项 | 类型 | 默认值 |
|--------|------|--------|
| projectName | string | - |
| projectDesc | string | - |
| packageName | string | - |
| frontendStack | 'vue3' \| 'react' | 'vue3' |
| backendStack | 'springboot' \| 'nestjs' | 'springboot' |
| database | 'mysql' \| 'postgres' \| 'h2' | 'mysql' |
| enableUniapp | boolean | false |
| enableDocker | boolean | true |

### .ai/architecture.md

```markdown
# 设备管理系统架构文档

## 项目概述
- 名称：equipment-system
- 类型：前后端分离 + 小程序
- 技术栈：Vue3 + Spring Boot + MyBatis Plus + uni-app

## 系统架构

### 分层架构
- API 层：提供 RESTful API
- Service 层：业务逻辑处理
- Mapper 层：数据访问
- Entity 层：数据模型

### 前端架构
- 状态管理：Pinia
- 路由：Vue Router 4
- UI：Element Plus

### 移动端架构
- 框架：uni-app
- 状态管理：Pinia
- UI：uni-ui

## 模块划分
- system: 系统管理（用户、角色、权限）
- equipment: 设备管理
- borrow: 借用管理
- repair: 维修管理
- report: 报表统计
```
