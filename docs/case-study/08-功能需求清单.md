## 二、核心引擎功能

### 2.1 Spec Engine（规范引擎）

| 功能 | 描述 | 优先级 | 案例关联 |
|------|------|--------|----------|
| **需求解析** | 解析 Markdown 需求文档，提取实体和功能 | P0 | 01-需求规格说明书 |
| **实体识别** | 自动识别文档中的实体及其属性 | P0 | 识别 User/Equipment 等实体 |
| **关系抽取** | 抽取实体间的关系（1:N, N:M） | P1 | User ↔ BorrowRecord |
| **API 文档生成** | 根据实体自动生成 API 文档 | P1 | 生成 EquipmentController |
| **规范版本管理** | 规范文档的版本控制和变更追踪 | P2 | specs/ 版本历史 |

**输入**：Markdown 需求文档
**输出**：结构化实体定义、API 设计

### 2.2 Template Engine（模板引擎）

| 功能 | 描述 | 优先级 | 案例关联 |
|------|------|--------|----------|
| **项目模板** | 生成标准化项目结构 | P0 | 02-项目初始化 |
| **模块模板** | 生成标准化模块结构 | P0 | backend module create |
| **页面模板** | 生成标准化 Vue 页面 | P0 | frontend page create |
| **组件模板** | 生成可复用组件 | P0 | EquipmentForm.vue |
| **API 模板** | 生成前后端 API 调用代码 | P0 | uniapp api generate |
| **部署模板** | 生成 Docker/K8s 配置 | P1 | 07-测试与部署 |

### 2.4 Repair Engine（修复引擎）

| 功能 | 描述 | 优先级 | 案例关联 |
|------|------|--------|----------|
| **错误检测** | 自动检测编译/测试错误 | P0 | 07-测试失败检测 |
| **根因分析** | 分析错误根本原因 | P0 | 07-定位权限校验问题 |
| **自动修复** | 根据错误类型自动修复 | P1 | 07-自动修复测试失败 |
| **回归验证** | 修复后自动运行测试验证 | P0 | 07-验证测试通过 |
| **修复历史** | 记录修复历史供学习 | P2 | .ai/repair-history.md |

**修复策略**：

```typescript
interface RepairStrategy {
  // 错误类型
  errorType: 'compilation' | 'test' | 'runtime' | 'logic'
  
  // 检测规则
  detect: (error: Error) => boolean
  
  // 修复方案
  fix: (error: Error, context: CodeContext) => FixResult
  
  // 验证方式
  verify: (fix: FixResult) => Promise<boolean>
}
```

## 五、AI 能力矩阵

### 5.1 按场景分类

| 场景 | 能力 | 说明 |
|------|------|------|
| **代码生成** | 自然语言 → 代码 | 根据描述生成代码 |

| **文档生成** | 代码 → 文档 | 生成 API 文档 |
